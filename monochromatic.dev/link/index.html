Formats

We support the following formats:

HTML

Files ending in .html are treated as HTML by default.

Declare a different file format by inserting an HTML element <x-variables> format = "md" </x-variable>.

Element content follows toml syntax.

Example: Specifying a file having an extension of .html to be GitHub flavored markdown. 


<x-variables> format = "md" </x-variable>

Flexible formatting of HTML

There are several ways to write an HTML document:

Writing content directly

.

Writing content inside the <article> tag

Advantage over writing content directly:
  1. You can write multiple articles in one page.

Default auto applied transforms:

Document starting with <article>

  1. <wrapper-block> is wrapped against the whole document.

  2. A "in this article" sidebar is generated:

    
  <aside id="In_this_article" >
  <h1>
    In this article
  </h1>

  <ol>
  <li>
    <a href="#Article_Name__First_h2" >First h2</a>
  </li>

  <li>
    <a href="#Article_Name__Second_h2" >Second h2</a>

    <ol>
    <li>
      <a href="#Article_Name__Second_h2__First_h3" >First h3</a>
    </li>

    <li>
      <a href="#Article_Name__Second_h2__Second_h3" >Second h3</a>
    </li>
    </ol>
  </li>
  </ol>
  </aside>

Document does not start with <article> tag

  1. <h1> tag is removed from the document and put into header.
  2. <article> tag is wrapped around the document

. . .before continuing to apply other transforms.

Tags staring with x-

These tags are further transformed into appropriate tags:
  1. <x-summary> found in <section>s are transformed into the appropriate heading levels according to their parent heading but given styling of <p>

Markdown (GitHub flavored markdown)

Files ending in .md are treated as GitHub flavored markdown by default.

Declare a different file format by inserting an HTML element <x-variables> format = "html" </x-variable>.

Element content follows toml syntax.

Example: Specifying a file having an extension of .md to be HTML. 


<-- format = "html" -->
